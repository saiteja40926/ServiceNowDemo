Executive Summary
What SearchPulse Does

SearchPulse is a lightweight agentic system that improves enterprise search trust by passively observing user behavior during search sessions, scoring article reputation from those signals, and surfacing two key outputs: an AI Summary Confidence Score and Trust Badges on knowledge article search results. No ML training. No new infrastructure. Runs entirely on ServiceNow scripting.

📊 What It Captures

  • Thumbs up/down on AI summary
  • Scroll past AI summary into results
  • Article click-through
  • Return to search after clicking
  • Query reformulation

🏷️ What It Shows

  • High Confidence — green badge
  • Medium Confidence — amber badge
  • Low Confidence — red badge
  • Frequently Helpful
  • Updated Recently
  • May Be Outdated

⚙️ What It Automates

  • Daily reputation score recalculation
  • Session outcome classification (A–F)
  • Content quality alerts (score < 40)
  • KB owner task creation
  • Analytics dashboard refresh

Reputation Calculator — Scoring Logic

Daily Job: "SearchPulse Reputation Calculator" — Runs at 02:00 every night

✅ Positive Outcomes

OutcomePointsMeaning
A+3Summary satisfied user
C+1Drilled down successfully

❌ Negative Outcomes

OutcomePointsMeaning
D-5Clicked result, came back
E-10Reformulated query
F-15Rejected AI summary
Formula: new_score = clamp( current_score + Σ(outcome_weights), 0, 100 )
Default: All articles start at score = 50 (neutral)
Trigger: Content Quality Monitor fires when score drops below 40

Demo Seed Data Script

Run in Scripts - Background (right-click Navigator > Scripts - Background) to pre-populate realistic demo data before the leadership presentation:

// SearchPulse Demo Data Seeder
// Run in: System Definition > Scripts - Background
// Purpose: Creates sample reputation records for leadership demo

var DEMO_DATA = [
    { title: 'Expense Guide', score: 82, pos: 47, neg: 3, sessions: 55 },
    { title: 'Finance FAQ',   score: 61, pos: 28, neg: 8, sessions: 40 },
    { title: 'Legacy Expense Policy', score: 38, pos: 5, neg: 22, sessions: 30 }
];

DEMO_DATA.forEach(function(d) {
    var kbGR = new GlideRecord('kb_knowledge');
    kbGR.addQuery('short_description', 'CONTAINS', d.title);
    kbGR.setLimit(1);
    kbGR.query();
    if (!kbGR.next()) {
        gs.print('SKIP: article not found - ' + d.title);
        return;
    }
    var repGR = new GlideRecord('x_searchpulse_article_reputation');
    repGR.addQuery('article', kbGR.getUniqueValue());
    repGR.setLimit(1);
    repGR.query();
    var isNew = !repGR.next();
    if (isNew) { repGR = new GlideRecord('x_searchpulse_article_reputation'); repGR.initialize(); }
    repGR.setValue('article',          kbGR.getUniqueValue());
    repGR.setValue('reputation_score',  d.score);
    repGR.setValue('positive_signals',  d.pos);
    repGR.setValue('negative_signals',  d.neg);
    repGR.setValue('sessions_count',    d.sessions);
    repGR.setValue('last_calculated',   new GlideDateTime().getValue());
    isNew ? repGR.insert() : repGR.update();
    gs.print('SEEDED: ' + d.title + ' -> score=' + d.score);
});
gs.print('SearchPulse demo data seeding complete.');

Technical Notes & Constraints

✅ What This Covers

  • All 3 tables with correct field types and choice lists
  • 4 Script Includes matching spec names exactly
  • GlideAjax-compatible SignalAPI (client_callable=true)
  • Outcome logic A–F per specification
  • Score weights A(+3), C(+1), D(-5), E(-10), F(-15)
  • Badge rules per specification
  • Confidence bands: ≥70 High, 45-69 Medium, <45 Low
  • Default reputation_score = 50
  • Score clamped 0–100
  • Flow trigger + Business Rule when score < 40
  • 4 analytics reports + PA Dashboard

⚠️ Post-Import Notes

  • Tables extend sys_metadata base (no task dependency)
  • PA Dashboard widgets require Performance Analytics plugin
  • Flow Designer hub flow requires Flow Designer plugin
  • Business Rule is the reliable fallback for the quality monitor
  • UI Script uses window.GlideAjax — must load on SP pages
  • sendBeacon fallback uses a stub endpoint; configure Scripted REST if needed
  • Seed demo data before leadership presentation
  • sys_id values are deterministic — safe to re-import
